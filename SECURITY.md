# Security Policy

## Supported Versions

Security updates are applied to the latest version on the main branch.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Report vulnerabilities by email: adejaredaniel12@gmail.com

Please include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation if available

You can expect an acknowledgment within 72 hours.

## Secret Handling

- Never commit real credentials to the repository.
- Use .env.sample files for documentation.
- Rotate exposed credentials immediately.
